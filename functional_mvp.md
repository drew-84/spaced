# SPACED — MVP Spec (v1)

## Core Idea

SPACED is a platform that enables users to access private physical space on demand, based on time.

Instead of rigid systems like hotels, leases, or memberships, SPACED allows users to instantly book nearby spaces for short durations, with the flexibility to extend or leave at any time.

The goal is to make physical space behave like an on-demand resource.

## Initial Wedge

The MVP focuses on:

👉 instant access to nearby private spaces for short durations (30–60 minutes), driven by time-sensitive, discreet use cases

## Core User Moment

A user needs a private space immediately, with minimal friction and high discretion.

## Dominant Use Case

- short-duration private encounters (primary)
- secondary: rest, temporary privacy, short escape

## Time Model

- minimum: 30 minutes
- standard options: 30 or 60 minutes
- extensions: +15 minute increments

This allows users to start with a short booking and extend in real time if needed.

## Type of Space

- private rooms
- studios
- 1-bedroom apartments

Supply is flexible and determined by availability, location, and price.

## Core Value Proposition

Private space, on demand, near you, for as little time as you need — with no commitment and high discretion.

👉 With Uber you summon trips. With SPACED you summon space.

## Core Transaction Loop

1. A host lists a space with availability
2. A user discovers nearby spaces
3. The user selects a duration
4. The user books and pays
5. The user receives access instructions
6. The session takes place
7. The session ends or is extended

## Minimum Product Requirements

- user accounts
- host listing creation
- nearby space discovery (map or list)
- duration selection
- booking + payment
- access delivery (code / instructions)
- booking confirmation

## Access Model (MVP Constraint)

Access must not depend on host presence.

Accepted access types:

- digital lock
- lockbox (key stored securely with code access)
- concierge or front desk

## Cleaning & Turnover (MVP)

Each booking includes a buffer period between sessions to allow for turnover.

- buffer: 15–30 minutes (configurable)
- host is responsible for ensuring the space is in acceptable condition
- host may use personal or third-party cleaning methods

The platform does not coordinate cleaning in MVP.

The system prioritizes:

👉 reliable access over perfect standardization

## Key Product Constraints

- ultra-fast interaction (low friction)
- immediate availability visibility
- clear pricing upfront
- discreet user experience
- simple and reliable access

## MVP Success Definition

👉 A user successfully books a nearby private space for a short duration, receives access, and completes the session without significant friction.
